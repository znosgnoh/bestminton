# US-15: Prevent Duplicate Expense Submission

**Epic:** Error Handling & Edge Cases

**As a** team captain,
**I want to** be prevented from accidentally submitting the same expense twice,
**so that** Splitwise doesn't record duplicate charges.

## Acceptance Criteria

- [ ] The "Sync to Splitwise" button is disabled immediately after being clicked and remains disabled during the API call.
- [ ] After a successful sync, the button is replaced by a success state (e.g., a checkmark), preventing re-submission.
- [ ] To start a new session, the captain must explicitly reset the form.
