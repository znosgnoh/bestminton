# US-31: Management Match List — Upcoming / Past Tabs

**Epic:** Management UX

**As a** captain,
**I want** the management match list split into Upcoming and Past tabs,
**so that** I can focus on scheduling future sessions without past matches cluttering the view.

## Acceptance Criteria

- [x] The match list in `/management` has two tabs: **Upcoming** and **Past**.
- [x] Each tab label shows the count of matches in that category (e.g. "Upcoming (3)").
- [x] **Upcoming** tab: shows matches scheduled from now onward, sorted ascending (soonest first).
- [x] **Past** tab: shows matches before now, sorted descending (most recent first).
- [x] Both orderings follow the "nearest date from today" principle — upcoming counts up from today, past counts back from today.
- [x] Creating a new match updates the count on the correct tab immediately without a page reload.
- [x] Deleting a match removes it from the active tab immediately.
- [x] Tab styling matches the homepage Upcoming / Past tabs (emerald underline for active, gray for inactive).
