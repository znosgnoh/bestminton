# US-10: Sync Expense via Backend Proxy

**Epic:** Splitwise Sync

**As a** team captain,
**I want to** push the calculated expense to Splitwise with one click,
**so that** all members are notified of what they owe without me manually entering data in Splitwise.

## Acceptance Criteria

- [ ] Clicking "Sync to Splitwise" sends a request to the Next.js Route Handler at `/api/splitwise`, not directly to Splitwise.
- [ ] The Route Handler attaches the server-side `SPLITWISE_API_KEY` and forwards the request to `POST https://secure.splitwise.com/api/v3.0/create_expense`.
- [ ] The API key is never exposed to the client/browser.
- [ ] A loading indicator is shown while the sync is in progress.
- [ ] On success, a confirmation message is shown (e.g., "Expense created successfully!").
