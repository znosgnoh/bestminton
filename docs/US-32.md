# US-32: Settle Match Restricted to Management Context

**Epic:** Access Control

**As a** captain,
**I want** the Settle Match section to appear only when I open a past match from the management page,
**so that** regular players viewing past matches on the homepage only see the attendance list, not cost and Splitwise controls.

## Acceptance Criteria

- [x] On the homepage (`/`), tapping a past match opens `/matches/[id]` without `?manage=1` — only match info and the registered players/guests list are shown.
- [x] On `/management`, each past match row has a clipboard icon button.
- [x] Clicking the clipboard icon navigates to `/matches/[id]?manage=1`.
- [x] The Settle section (`SettleForm`) — including cost entry, hours, paid-by, share breakdown, and Sync to Splitwise — is only rendered when `?manage=1` is present.
- [x] The `?manage=1` flag is read server-side from `searchParams` in the page component and passed as a prop to the client component (no client-side URL parsing).
- [x] Manually adding `?manage=1` to the URL in the browser does show the Settle section — this is intentional and consistent with the security-by-obscurity model.
