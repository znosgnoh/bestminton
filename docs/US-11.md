# US-11: Construct the Correct Splitwise Payload

**Epic:** Splitwise Sync

**As a** system,
**I want to** format the expense data in Splitwise's required flattened key-value format,
**so that** the API accepts the request without validation errors.

## Acceptance Criteria

- [ ] The payload includes `cost`, `description`, `group_id`, and `currency_code` fields.
- [ ] For each participant `i` (0-indexed), the payload includes:
  - `users__i__user_id`
  - `users__i__paid_share` (the payer's field equals `Total_Cost`; all others equal `0.00`)
  - `users__i__owed_share`
- [ ] `paid_share` and `owed_share` values are formatted as strings with two decimal places.
- [ ] The sum of all `owed_share` values equals `Total_Cost` exactly (rounding adjustment applied, per US-08).
- [ ] The payer's `paid_share` equals `Total_Cost`; all other members' `paid_share` equals `0.00`.
