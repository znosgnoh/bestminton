# US-16: Handle Single Participant Edge Case

**Epic:** Error Handling & Edge Cases

**As a** system,
**I want to** correctly calculate the split when only one member participates,
**so that** the sole participant is charged the full `Total_Cost`.

- [ ] The single member's owed share equals `Total_Cost` exactly.
- [ ] The rounding adjustment logic does not produce an incorrect result in this case.
- [ ] The sync payload is valid and accepted by Splitwise.
