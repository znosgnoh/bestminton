# US-13: Handle Splitwise API Errors

**Epic:** Error Handling & Edge Cases

**As a** team captain,
**I want to** receive a clear error message if the Splitwise sync fails,
**so that** I know the expense was not created and can take corrective action.

## Acceptance Criteria

- [ ] If the Route Handler receives a non-2xx response from Splitwise, it returns a structured error to the client.
- [ ] The UI displays a human-readable error message (e.g., "Sync failed: Invalid API key. Please check your configuration.").
- [ ] The captain can retry the sync without having to re-enter all session data.
- [ ] A network timeout or connection failure is also caught and surfaced as an error message.
