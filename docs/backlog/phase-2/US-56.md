# US-56: Protect Challenge Admin Actions with PIN

**Epic:** Admin Protection

**As a** captain,
**I want** Start and Resolve buttons gated by a PIN,
**so that** casual taps on the shared device don't alter match state.

## Acceptance Criteria

- [ ] `ADMIN_PIN` (or shared `CAPTAIN_PIN`) in `.env.local`; if unset, admin buttons work without gate (dev convenience), consistent with US-33 behavior.
- [ ] PIN prompt required before "Start Challenge" and "End Match & Resolve".
- [ ] Wrong PIN shows error; correct PIN unlocks admin controls for the session (`sessionStorage`, expires on tab close).
- [ ] **No** NextAuth, JWT, or user accounts introduced.
