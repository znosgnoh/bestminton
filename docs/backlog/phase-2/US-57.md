# US-57: Handle Empty or One-Sided Betting Pools

**Epic:** Edge Cases & Error Handling

**As a** system,
**I want** resolution to behave correctly when no one bet or only one side has bets,
**so that** the app never crashes or mis-allocates tokens.

## Acceptance Criteria

- [ ] Zero bets on both sides: challenge still completes; Elo updates; no token changes.
- [ ] Bets only on winning side: winners break even (stake returned); no net gain from an empty losing pool.
- [ ] Bets only on losing side: all losers forfeit stakes; forfeited tokens are burned (removed from circulation) when no winning-side bettors exist.
- [ ] UI on completed challenge explains "No bets placed" or "No payout — no one bet on the winner".
