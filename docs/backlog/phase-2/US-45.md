# US-45: Persist Bet Records

**Epic:** Data Foundation

**As a** system,
**I want** each bet linked to a challenge, bettor, predicted winner, and fixed amount,
**so that** pari-mutuel resolution can run deterministically.

## Acceptance Criteria

- [ ] `Bet` entity: `id`, `challengeId`, `bettorId`, `predictedWinnerId` (Side A or Side B representative player ID), `amount` (default `1`).
- [ ] Unique constraint: one bet per `(challengeId, bettorId)` while `PENDING` or `ACTIVE`.
- [ ] Bets are immutable once challenge status is `ACTIVE` or `COMPLETED`.
- [ ] Default bet amount is `1` token (1 drink); configurable constant, not per-user input in v1.
