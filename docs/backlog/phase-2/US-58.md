# US-58: Handle Invalid Challenge State and API Failures

**Epic:** Edge Cases & Error Handling

**As a** player,
**I want** clear feedback when something goes wrong,
**so that** the group isn't confused on court.

## Acceptance Criteria

- [ ] Cannot start an already `ACTIVE` or `COMPLETED` challenge (409 / disabled button).
- [ ] Cannot resolve a `PENDING` challenge (must start first).
- [ ] Cannot resolve an already `COMPLETED` challenge.
- [ ] Network/API errors show a retry-friendly banner; bet checkbox state rolls back on failed save.
- [ ] Concurrent bet updates: no duplicate bets per member (server enforces unique constraint).
- [ ] Member with insufficient `tokenBalance` to cover a new bet: checkbox disabled with tooltip "Insufficient tokens" (if negative balances disallowed in v1).
