# US-43: Persist Challenge Records

**Epic:** Data Foundation

**As a** system,
**I want** challenges stored with players, status, handicap, and winner,
**so that** match history and resolution are auditable.

## Acceptance Criteria

- [ ] `Challenge` entity: `id`, `playerAId`, `playerBId`, `status` (`PENDING` | `ACTIVE` | `COMPLETED`), `handicapPoints`, `winnerId` (nullable until completed), `createdAt`, `completedAt` (nullable).
- [ ] Status defaults to `PENDING` on creation.
- [ ] `winnerId` must be either `playerAId` or `playerBId` when set.
- [ ] Supports 1v1 (single ID per side); 2v2 extension covered in US-44.
