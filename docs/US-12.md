# US-12: Handle Empty Attendance

**Epic:** Error Handling & Edge Cases

**As a** team captain,
**I want to** be prevented from syncing when no members are checked,
**so that** I don't accidentally create a meaningless Splitwise expense.

## Acceptance Criteria

- [ ] The "Sync to Splitwise" (and "Review") button is disabled when zero members are checked.
- [ ] An inline message explains why the button is disabled (e.g., "Select at least one participant.").
