# US-08: Handle Rounding Discrepancies

**Epic:** Cost Calculation & Review

**As a** system,
**I want to** automatically correct floating-point rounding errors in the split,
**so that** the Splitwise API does not reject the expense due to a mismatched total.

## Acceptance Criteria

- [ ] After computing all `Owed_i` values, the app sums them and compares to `Total_Cost`.
- [ ] Any discrepancy (e.g., ±0.01, ±0.02) is applied as an adjustment to the first participating member's owed share.
- [ ] The adjusted total always equals `Total_Cost` exactly (no floating-point remainder).
- [ ] The adjustment is invisible to the user in the UI (it happens silently before sync).
